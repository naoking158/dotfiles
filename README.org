* Dotfiles

Nix (Home Manager + Flakes) + chezmoi + mise によるクロスプラットフォーム開発環境。

** Architecture

#+begin_example
chezmoi          -- dotfiles 管理 + Nix 設定の展開・自動適用トリガー
Nix Home Manager -- CLI ツールの宣言的管理 (git, ripgrep, fzf, etc.)
mise             -- 言語ランタイム管理 (Node.js, Go, Ruby)
Homebrew         -- macOS GUI アプリ (cask) のみ
#+end_example

#+begin_example
~/.config/
  home-manager/    ... Nix 設定 (flake.nix, home.nix, linux.nix, darwin.nix)
  mise/config.toml ... 言語ランタイム設定
  zsh/
    .zshenv        ... 環境変数 (XDG, EDITOR, LANG)
    .zshrc         ... 対話シェル設定 (sheldon, nonlazy, lazy)
#+end_example

** Bootstrap (新規マシン)

4 ステップで完了する。

*** Step 1: Determinate Nix のインストール

#+begin_src shell
curl -fsSL https://install.determinate.systems/nix | sh -s -- install
#+end_src

インストール後、新しいシェルを開くか =. /nix/var/nix/profiles/default/etc/profile.d/nix-daemon.sh= で PATH を通す。

*** Step 2: chezmoi でリポジトリを取得

#+begin_src shell
nix shell nixpkgs#chezmoi -c chezmoi init git@github.com:naoking158/dotfiles.git
#+end_src

*** Step 3: chezmoi apply (dotfiles + Nix パッケージの一括適用)

#+begin_src shell
nix shell nixpkgs#chezmoi -c chezmoi apply
#+end_src

これにより以下が実行される:
- dotfiles の展開 (.zshenv, .zshrc, lazy.zsh, git, tmux, etc.)
- Nix 設定ファイルの展開 (~/.config/home-manager/)
- mise 設定の展開 (~/.config/mise/config.toml)
- =run_once_after_cachix-use.sh= が自動実行され、 =nix-community= cachix を =/etc/nix/nix.custom.conf= に登録する (対話実行時のみ。 sudo パスワードが要求される)
- =run_onchange_nix-apply.sh= が自動実行され =home-manager switch= で CLI ツールがインストールされる (emacs-overlay 経由の Emacs はバイナリキャッシュから取得される)

初回は Nix パッケージのダウンロードに数分かかる。

*Note*: cachix のセットアップで sudo を求められるのは初回のみ。以後は =/etc/nix/nix.custom.conf= の存在を見て skip される。

非対話で =chezmoi apply= を実行した場合 (sudo 認証キャッシュなし & tty 無しの場合) は cachix セットアップが skip され警告が出る。その場合は以下を 1 度だけ手動実行:

#+begin_src shell
sudo /nix/var/nix/profiles/default/bin/nix run nixpkgs#cachix -- use nix-community
#+end_src

実行後、再度 =chezmoi apply= しても cachix スクリプトは skip されるが、必要なら =chezmoi state delete-bucket --bucket=scriptState= でリセットして再試行できる。cachix を有効化しないと emacs-overlay 経由の Emacs はソースビルド (時間がかかるが正しく動作はする)。

*** Step 4: 言語ランタイムのインストール

#+begin_src shell
mise install
#+end_src

Node.js, Go 等がインストールされる。

** 通常運用

*** 設定の変更を適用

#+begin_src shell
chezmoi apply
#+end_src

Nix 設定ファイル (flake.nix, home.nix, linux.nix, darwin.nix) に変更があれば =home-manager switch= が自動実行される。変更がなければスキップされる。

*** パッケージの追加

=~/.local/share/chezmoi/dot_config/home-manager/home.nix.tmpl= の =home.packages= にパッケージを追加し、 =chezmoi apply= を実行する。

*** nixpkgs の更新

#+begin_src shell
cd ~/.config/home-manager && nix flake update
#+end_src

その後 =chezmoi apply= でハッシュ変化を検知し =home-manager switch= が再実行される。

** Rollback

*** Home Manager の世代を切り戻し

#+begin_src shell
home-manager generations
# 出力される世代パスから activate を実行
/nix/store/xxx-home-manager-generation/activate
#+end_src

*** chezmoi の変更を元に戻す

#+begin_src shell
cd ~/.local/share/chezmoi
git checkout main
chezmoi apply
#+end_src

** OS 固有の設定

| 設定 | macOS | Pop OS (Linux) |
|------+-----------+----------------|
| flake entry | =<user>@macos= | =<user>@popos= |
| homeDirectory | =/Users/<user>= | =/home/<user>= |
| Homebrew | cask (GUI) のみ | 使用しない |
| genericLinux | - | =true= |

OS 固有のパッケージは =darwin.nix.tmpl= / =linux.nix.tmpl= に追加する。

** Linux (Wayland / Cosmic DE) 固有のセットアップ

xremap (キーリマッパー) と =emacs-lsp-proxy= は home-manager 経由で宣言的に管理される。Pop!_OS 上の Cosmic DE (=cosmic-comp=) を前提とし、xremap-flake は =services.xremap.withCosmic = true= で =xremap-cosmic= バリアントを選択する。

*** systemd user 環境変数の確認

Cosmic DE の =cosmic-session= は systemd user セッションへ =WAYLAND_DISPLAY= / =XDG_CURRENT_DESKTOP= を引き渡し =graphical-session.target= を立ち上げる。xremap.service は =WantedBy=graphical-session.target= で待機しているため、ログイン直後に自動起動する想定。

確認コマンド:

#+begin_src shell
systemctl --user show-environment | grep -E '(WAYLAND_DISPLAY|XDG_CURRENT_DESKTOP)'
systemctl --user status graphical-session.target
#+end_src

もし環境変数が立っていない (xremap が起動できない) 場合は、Cosmic 側の autostart で以下を 1 度実行する保険を入れる:

#+begin_src shell
systemctl --user import-environment WAYLAND_DISPLAY XDG_CURRENT_DESKTOP DISPLAY
systemctl --user start xremap
#+end_src

*** 動作確認コマンド

#+begin_src shell
# xremap.service の状態
systemctl --user status xremap
journalctl --user -u xremap

# emacs-lsp-proxy バイナリが PATH に出ているか
which emacs-lsp-proxy

# uinput / input グループの確認 (run_once_after_uinput-setup.sh が完了済みか)
getent group input | grep -qw "$USER" && echo "ok: $USER in input group"
test -f /etc/udev/rules.d/99-input.rules && echo "ok: udev rule exists"
test -f /etc/modules-load.d/uinput.conf && echo "ok: modules-load.d entry exists"
lsmod | grep -qw uinput && echo "ok: uinput module loaded"
#+end_src

=run_once_after_uinput-setup.sh= は input グループ追加・udev ルール作成・=modprobe uinput= を冪等に行う。完了後はセッションへの input グループ反映のため一度ログアウト→再ログインすること。NixOS 上ではテンプレートガードにより no-op となる。
